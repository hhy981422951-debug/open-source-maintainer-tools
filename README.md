# Maintainer Pulse

Maintainer Pulse is a small, local-first triage cockpit for open-source maintainers. Paste an issue, pull request summary, or support question, and the page suggests labels, priority, effort, a maintainer checklist, a first reply, and a release-note draft.

The app runs entirely in the browser. It does not require an API key, a server, or a build step.

## Features

- Rule-based issue and pull request classification
- Suggested labels such as bug, docs, feature, question, release, and windows
- Priority and effort meters
- Draft maintainer response
- Draft release-note line
- Local scan history stored only in the browser
- Markdown report copy button

## How to Use

Open `index.html` in a browser, paste issue text, or use one of the built-in samples.

## Why This Exists

Small open-source projects often need lightweight maintenance workflows before they need heavier bots or hosted automation. This project is a practical starting point for exploring maintainer tools that stay simple, transparent, and easy to adapt.
